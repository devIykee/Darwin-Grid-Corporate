SETUP:

- ws.Server attached to the same http.Server instance

TWO TYPES OF WEBSOCKET CLIENTS:
1. BROWSER SPECTATORS — connect to WS /ws/spectate
   - They only RECEIVE state broadcasts
   - They never send messages
   - On connect, immediately send them the full world snapshot

2. EXTERNAL AGENTS — connect to WS /ws/agent-connect
   - Must send { type: "AUTH", ws_token: "..." } as first message
   - If token not found in AGENTS map, close the connection with code 4001
   - After auth, they receive their personal agent state every tick
   - They can send action payloads (see Section 4G)

TICK LOOP (setInterval every TICK_INTERVAL_MS):
  For each agent (in AGENTS, not bankrupt):
    1. agent.debtBalance += 0.001
    2. If agent has currentPath steps remaining:
         pop next step from currentPath
         agent.position = next step
         agent.walletBalance -= TRAVEL_COST_PER_TILE
    3. If agent has arrived at a resource tile AND status === "mining":
         call mineResource(agent) — see below
    4. Check bankruptcy
    5. If agent is "idle" AND not epochEnded:
         call orchestrateAgent(agent) — HTTP POST to agent-orchestrator
  After all agents processed:
    increment tickCount
    check epoch end condition
    broadcast world state to ALL spectator WebSocket clients

mineResource(agent):
  - Get the resource at agent.position from GRID
  - If resource exists and not claimed:
      Mark resource as claimed
      agent.walletBalance += resource.value
      agent.status = "idle"
      Remove resource from grid
      If agent has currentContract:
          Call circle-settlement POST /settle-contract (see Section 6C)
          agent.reputation += 2
          agent.currentContract = null
      Broadcast { type: "RESOURCE_MINED", agent_id, resourceId, position }

orchestrateAgent(agent):
  - Build the input payload (see Section 5B)
  - HTTP POST to agent-orchestrator at http://localhost:3002/orchestrate
  - On response, apply the returned decision to the agent (see Section 5C)
  - This is async — do not block the tick loop. Fire and forget with error handling.

BROADCAST FORMAT (sent to all spectator WebSocket clients every tick):

{

  epochEnded: boolean,

5E. Applying Orchestrator Decisions (in world-server agentManager)
─────────────────────────────────────────────

When world-server receives a decision from the orchestrator:

If action === "MINE":
  - Find the nearest unclaimed resource on the grid
  - Use A* to compute path from agent.position to resource.location
  - Set agent.currentPath = computed path
  - Set agent.status = "mining"

If action === "CREATE_COMPANY":
  - HTTP POST to circle-settlement /create-company
  - Body: { agent_id, company_name: decision.company_name, initialCapital: agent.walletBalance * 0.5 }
  - On success: agent.companyId = returned companyId
  - Deduct initialCapital from agent.walletBalance
  - Console log: [TX] create-company <name> → <companyId>

If action === "POST_CONTRACT":
  - Validate contract_details is not null
  - HTTP POST to circle-settlement /escrow-contract
  - Body: { companyId: agent.companyId, contractId: uuid(), reward: contract_details.reward }
  - On success: add contract to global openContracts list
  - Console log: [TX] escrow-contract <contractId> amount=<reward>

If action === "ACCEPT_CONTRACT":
  - Find the contract by contractId from open_contracts (ignore contracts posted by same agent)
  - Set agent.currentContract = contractId
  - Set contract.status = "in_progress"
  - Compute path to contract.targetLocation
  - Set agent.currentPath = path
  - Set agent.status = "contracted"

If action === "WAIT":
  - Do nothing. Agent stays idle.

    removeResourceFromScene(msg.resourceId);

SECTION 9: ERROR HANDLING RULES
═══════════════════════════════════════════════════════════

RULE 1: Circle API errors must NEVER crash any service.
  Wrap every Circle SDK call in try/catch.
  On error: log the error, return HTTP 500 with { error: message }, continue running.

RULE 2: Anthropic API errors must fall back gracefully.
  If the LLM call fails or returns unparseable JSON:
  Return { action: "WAIT", reasoning: "API error fallback", target: null, contract_details: null, company_name: null }

RULE 3: WebSocket disconnects must be handled.
  If a browser spectator disconnects, remove them from the broadcast list silently.
  If an external agent disconnects, set their status to "idle" and keep their state.
  They can reconnect with the same ws_token.

RULE 4: Do not crash on missing agents.
  If settle-contract references an agent_id not in AGENTS, log warning and continue.

RULE 5: Tick loop must never throw uncaught exceptions.
  Wrap the entire tick body in try/catch. Log errors. Continue to next tick.
